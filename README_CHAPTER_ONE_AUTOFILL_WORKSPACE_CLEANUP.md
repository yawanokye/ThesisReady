# Chapter One Auto-Fill and Workspace Cleanup Update

This update improves the thesis writing workspace so students are not asked to repeat information that already exists in an uploaded Chapter One or introduction chapter.

## What changed

- Chapter One, earlier chapters, or a complete existing work can be uploaded near the project setup area.
- Uploaded Chapter One is used to auto-extract available alignment details such as study context, problem background, objectives, research questions, hypotheses, variables and theory/framework extracts.
- Blank frontend fields no longer overwrite alignment information extracted from uploaded chapters.
- Chapter Two and later drafts use uploaded earlier chapters for alignment checks before asking for extra manual inputs.
- The workspace now treats missing information as confirmation placeholders, not hard stops, except for basic academic-integrity and responsibility confirmations.
- Long PhD and doctoral chapters continue to use staged drafting and final coherence checking.
- The workspace layout was reorganised with a clearer setup path, friendlier guidance cards and a more visible Chapter One upload area.
